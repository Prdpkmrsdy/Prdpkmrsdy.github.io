Automated Test Jig for Transformer Testing (LabVIEW)

Final Year Project · PSG Institute of Technology and Applied Research · Mar 2021

Built a LabVIEW-based automated testing setup to reduce manual effort, improve repeatability, and simplify data logging for routine transformer tests. The approach targets quicker execution and easier handling of test data in manufacturer premises. (Project report: “Automated Test Jig for Transformer Testing Using LabVIEW”).

LabVIEW NI myDAQ / DAQ Plug-and-Play Logging Open Circuit Test Short Circuit Test Test Automation

Impact

  • Reduced overall testing time from 34 hours to 19 hours (as implemented at Padmavahini Transformers Pvt Ltd).
  • Lower operator dependency and improved repeatability through automated measurement + calculation flow.
  • Improved data handling: logging + display through LabVIEW front panel for quick decision-making.

What I built

A “virtual test jig” implemented in LabVIEW that acquires analog measurements (via CT/PT and DAQ), computes relevant parameters, and displays results on a LabVIEW front panel. The intent is to streamline routine transformer testing and simplify data storage/handling.

CT/PT Sensing DAQ → PC Front Panel UI

Testing Scope

The project focuses on automating routine test procedures and presenting computed values directly in LabVIEW. The report highlights OC and SC test implementation and notes future extension to more complex tests.

  • Implemented: Open Circuit Test and Short Circuit Test measurement + calculations + front-panel display.
  • Planned/Extendable: MBT, Vector Group, DVDF, Voltage Ratio (VR), and other routine tests.

System Architecture

  • Measurements captured using CT (current) and PT (voltage) sensing, digitized via DAQ, processed on a PC/Laptop.
  • LabVIEW handles acquisition, computation, and UI display (front panel).
  • Designed for cleaner workflows and better traceability of results (logging + storage ready).
DAQ CT / PT LabVIEW VI Dataflow

Key Deliverables

  • LabVIEW VI for OC/SC tests (measurement + computation).
  • Front panel UI showing calculated parameters and readings.
  • Test workflow that reduces manual steps and improves repeatability.
