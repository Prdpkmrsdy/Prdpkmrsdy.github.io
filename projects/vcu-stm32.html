Automotive Vehicle Control Unit (VCU) for SAE Electric Vehicle

Central embedded controller integrating power, communication, sensing, and safety logic for an SAE electric vehicle.

Designed and developed an automotive-grade Vehicle Control Unit (VCU) that acts as the central brain of an SAE electric vehicle. The system integrates power management, CAN communication, sensing, safety-critical I/O, and user mode control into a single embedded platform.

The design emphasizes robust power architecture, fault isolation, and diagnostics-ready firmware, making it suitable for real-world EV applications beyond a student prototype.

STM32L496RE CAN (Dual) Power Sequencing Opto-Isolated I/O GPS/GSM ICM-20600 IMU SD Logging ATSHA204A
What This VCU Controls
  • Vehicle power-up and shutdown sequencing
  • CAN communication with motor controller, BMS, and dashboard
  • Drive mode selection (Eco / Normal / Sport / Reverse)
  • Throttle signal conditioning and output control
  • Vehicle safety inputs and diagnostics
System Architecture Overview
  • Power Architecture: 12V input, buck + LDO regulation, Li-ion backup, seamless rail switching
  • Core Processing: STM32L496RE with CAN, UART, ADC/DAC, GPIO
  • Communication: Dual CAN transceivers + GPS/GSM module (MC60 series)
  • Sensing: IMU, voltage/current monitoring, battery and board temperature sensing
  • Safety I/O: Opto-isolated vehicle inputs/outputs for fault protection
Engineering Challenges Addressed
  • Designing fault-tolerant power sequencing with backup operation
  • Protecting MCU from vehicle electrical noise using opto-isolation
  • Managing multiple communication interfaces reliably (CAN + GPS/GSM)
  • Building a firmware structure suitable for diagnostics and future scaling
